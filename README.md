# ComfyUI Metadata Rescue

Recover the generation settings hidden inside your favorite AI images.

**ComfyUI Metadata Rescue** is a planned ComfyUI custom-node extension for reading embedded generation metadata from original images and turning it into something humans can actually use.

It is designed for the painful but extremely common situation where you generate an image you love, forget to save the workflow or settings, and later discover that the only copy left is the exported PNG.

Instead of handing you a giant wall of JSON and wishing you luck, Metadata Rescue is intended to recover and expose the useful parts:

- Positive and negative prompts
- Model, CLIP, and VAE filenames
- Seed, steps, CFG, sampler, scheduler, and denoise settings
- LoRA stack and strengths
- Raw ComfyUI `Prompt` JSON
- Raw ComfyUI `Workflow` JSON
- A reproduction-safe workflow copy with the seed mode changed from `randomize` to `fixed`

## Status

> **Early development / proof of concept**

The metadata parsing logic has been tested against original Krea 2 / ComfyUI PNG metadata containing both embedded `Prompt` and `Workflow` graphs.

The current goal is to turn that tested recovery logic into a clean, useful ComfyUI node with an image loader and practical text outputs.

## Why this exists

Many metadata tools can tell you that an image has a `Prompt` or `Workflow` field, but they stop short of making that information usable.

A Comfy workflow can contain:

- Multiple loaders
- LoRA stacks
- Custom-node settings
- Text encoders
- KSampler settings
- Seed widgets set to `randomize`
- A full canvas workflow that is difficult to inspect manually

Metadata Rescue is intended to bridge that gap:

```text
Original image
      ↓
Embedded metadata
      ↓
Readable settings + reusable workflow exports
```

## Planned Main Node

### `Load & Read Metadata`

A single node that loads an image and exposes recovered metadata through normal ComfyUI connectors.

### Planned input

| Input | Description |
|---|---|
| `image` | Original image with embedded metadata |
| `seed_patch_mode` | Leave the workflow untouched or convert recoverable sampler seed modes to `fixed` |

### Planned outputs

| Output | Description |
|---|---|
| `positive_prompt` | Recovered positive prompt text |
| `negative_prompt` | Recovered negative prompt text |
| `readable_summary` | Human-readable generation recipe |
| `raw_prompt_json` | Embedded ComfyUI API prompt graph |
| `raw_workflow_json` | Embedded ComfyUI canvas workflow |
| `fixed_workflow_json` | Workflow copy made safer for exact reproduction |
| `model` | Diffusion model filename |
| `clip` | CLIP / text-encoder filename |
| `vae` | VAE filename |
| `seed` | Recovered numeric seed |
| `steps` | Sampling step count |
| `cfg` | CFG value |
| `sampler` | Sampler name |
| `scheduler` | Scheduler name |
| `denoise` | Denoise value |
| `loras` | Readable LoRA list with strengths |
| `status` | Metadata detection result or warning |

## Seed Recovery

One of the biggest gotchas in recovered Comfy workflows is that an image can preserve the seed used for that render while the original workflow remains set to:

```text
randomize
```

That means dragging the recovered workflow back into ComfyUI may produce a completely different image on the next queue.

Metadata Rescue will optionally create a reproduction-safe workflow copy that changes recognized KSampler seed modes:

```text
randomize → fixed
increment → fixed
decrement → fixed
```

The recovered numeric seed remains unchanged.

## Supported Metadata Targets

### Initial focus

- Original PNG files generated by ComfyUI
- Original PNG files generated through Krea 2 / ComfyUI workflows
- Standard PNG metadata chunks:
  - `tEXt`
  - `zTXt`
  - `iTXt`
- Embedded ComfyUI:
  - `Prompt`
  - `Workflow`

### Planned future support

- Automatic1111 `Parameters`
- Forge metadata
- WEBP metadata
- JPEG metadata
- Optional ExifTool fallback for unusual or non-PNG formats
- Batch metadata recovery outside ComfyUI

## Important Limitations

This extension can only recover metadata that still exists in the image file.

Metadata is commonly removed by:

- Discord uploads
- X/Twitter uploads
- Image-hosting sites
- Screenshots
- Some image editors
- Some upscalers and converters
- Re-saving an image through software that strips PNG text chunks

For best recovery odds, always use the original locally saved image.

Also, matching a recovered seed does not always guarantee pixel-perfect reproduction. Exact results can still depend on:

- The exact model and LoRA files
- Model or LoRA version changes
- ComfyUI version
- Custom-node version
- Sampler implementation changes
- Workflow changes
- Hardware and backend differences
- Nondeterministic nodes or effects

## Intended Installation

Once the extension is released, installation should follow the normal ComfyUI custom-node pattern:

```text
ComfyUI/
└── custom_nodes/
    └── ComfyUI-Metadata-Rescue/
```

Planned installation options:

### ComfyUI Manager

Search for:

```text
ComfyUI Metadata Rescue
```

### Manual installation

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/YOUR-USERNAME/ComfyUI-Metadata-Rescue.git
```

Then restart ComfyUI.

## Project Philosophy

This project is meant to be practical.

No one should have to manually dig through a 50,000-character metadata blob just to find:

```text
Which LoRA did I use?
What was the seed?
Why will this workflow not reproduce the image?
Where is the actual prompt?
```

The goal is simple:

> Load an image. Recover what matters. Rebuild what you can.
